# v0.2.0

SoftLoaf Trichrome v0.2.0 is the first stable desktop release. It covers the
core trichrome workflow: import grouped red/green/blue captures, preview the
composite, and export finished TIFF files with explicit color-space and bit
depth choices.

This stable label applies to the app workflow and packaging line. RAW camera
color correctness remains bounded by the provenance rules documented in the
repository; unsupported sensor classes and unverified camera transforms are not
silently promoted to full color-correct output.

## Changes

- Promoted the release line from beta to the first stable `v0.2.0` target.
- Fixed Frames selection export: Cmd/Ctrl+A in the Frames list now flows into
  the export window, defaults to `Selected frames`, and exports every selected
  complete frame without requiring each frame to be previewed first.
- Added an explicit `Selected frames` export range alongside `Current frame`
  and `All complete frames`.
- Centralized TIFF export writing in `src/export_writer.cpp`, including color
  encoding, integer quantization, ICC tagging, libtiff container writing, and
  Windows wide-character path handling.
- Added export integration coverage for color spaces, 8-bit/16-bit output,
  ICC presence, Unicode/deep paths, overwrite behavior, concurrent exports,
  and realistic 12MP 16-bit output.
- Added optional real-filesystem export tests for external volumes and SMB/NAS
  targets.
- Tightened monochrome RAW handling so monochrome RAW is treated as linear
  intensity and is not passed through an RGB camera matrix.
- Added Windows CI coverage on `windows-2022` using MSVC, Qt 6.8.3, and the
  pinned Windows vcpkg dependency manifest.
- Updated release automation defaults and documentation for the stable
  `0.2.0` release.

## Known Limitations

- DNG import remains LibRaw-backed; DNG export is not implemented in this
  release.
- RAW color correctness does not claim universal RawTherapee parity. Remaining
  RAW compatibility and color boundaries are tracked in
  `docs/raw_correctness_report.md`.
- Windows SmartScreen may warn if Authenticode signing credentials are not
  configured for this release.
- Auto-update manifests such as Sparkle, WinSparkle, or `latest.json` are not
  implemented yet.

## Verification

Local macOS validation before tagging:

```text
cmake --build build --target softloaf_trichrome_app
ctest --test-dir build --output-on-failure
```

Observed result:

```text
100% tests passed, 0 tests failed out of 4
```

Windows 11 `303mini` validation before tagging:

```text
cmake --build build-windows --target softloaf_trichrome_app --config Release
portable package generated with Qt/OpenCV/LibRaw/Little CMS runtime files
```

Release verification still requires the GitHub Actions macOS and Windows
release workflows for the exact `v0.2.0` tag. Do not publish the draft release
until the workflow-produced assets and SHA256 files are present and verified.

## Packaging Notes

- macOS DMG packaging must pass bundle dependency resolution, code signing,
  notarization, stapling, Gatekeeper assessment, and macOS 15.0 `minos`
  compatibility checks.
- Windows release publishes both installer and portable zip packages with Qt,
  OpenCV, LibRaw, Little CMS, libtiff, and runtime dependencies.
- Windows release assets remain unsigned unless the workflow has Authenticode
  credentials configured.
