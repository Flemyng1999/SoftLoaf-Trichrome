## 更新内容

- 修复 Windows 下中文路径、空格路径和非 ASCII 用户目录可能导致导入、导出或缓存失败的问题。
- 统一 Qt `QString`、`QUrl` 和 `std::filesystem::path` 的转换边界：Windows 使用 UTF-16，macOS/Linux 使用 UTF-8。
- 避免在核心项目元数据、导入报告和 artifact 路径中使用平台本地窄字符串编码。
- 导出和缓存路径继续使用 Qt/path 原生 API，减少路径窄化风险。
- 工具链中 OpenCV 读写改为字节流 `imdecode` / `imencode`，避免 Windows OpenCV 窄路径问题。
- 新增带空格和中文字符路径的自动化测试。
- RAW 对照工具链新增 RT32 oracle、项目 RAW probe 和差异 CSV 工具；这些工具用于兼容性验证，不代表已完成所有相机 RAW 正确性覆盖。

## 注意事项

- 当前版本仍是 alpha 测试版，不建议用于正式归档生产流程。
- Windows 安装包和便携包如未配置 Authenticode 证书，首次运行可能出现 SmartScreen 提示。
- RAW 相机兼容性仍在扩展和验证中。
- ICC / 色彩管理仍在完善，ACES AP0 和 ACEScg/AP1 路径目前属于早期支持。
