# v0.2.0-beta.3

这次 beta 主要是一次 RAW 兼容性与 Windows 可用性更新。重点同步了
SoftLoaf Negative 近期验证过的 RAW 解码边界，同时整理了 Frames 面板的
桌面交互。

这仍然是 beta 版本。它适合继续测试 trichrome 导入、预览、TIFF 导出、
色彩管理输出和受保护的 RAW 工作流，但不代表已经完整覆盖所有相机 RAW，
也不声明 RAW 色彩正确性已经达到 RawTherapee parity。

## 主要更新

- 提升 RAW 导入兼容性，并同步 SoftLoaf Negative 最新一轮 RAW 验证结果。
- 新增 Phase One / Leaf `.IIQ` 原生导入支持。只要当前 LibRaw 可以打开并
  解包，就会进入普通 Bayer/CFA RAW 路径。
- 同步多个针对真实相机样本的 RAW 解码修正与提示：
  - Sony packed full-color RAW 处理。
  - Sony ILCE-1M2 lossless medium RAW 的 processed-output crop hint。
  - Canon EOS 70D mRAW/sRAW white-level hint。
  - Leica SL2 内置 dcraw-style matrix override。
- 新增基于真实 RAW fixture 的回归测试与 provenance 诊断，方便确认不同
  相机样本的 RAW 行为。
- 改善 Frames 面板操作：全选、取消全选、Shift 连续选择、点击空白处取消
  选择，以及单个 Frame 删除按钮。
- 改善 Windows 上的 Qt Quick 样式兼容性和打包流程，让自定义 QML 界面更
  稳定地加载。

## RAW 兼容性

- Phase One / Leaf `.IIQ` 现在可以作为普通 LibRaw Bayer/CFA RAW 导入和
  解码。
- 更新 RAW policy / provenance identity，避免 cache 或 artifact 静默复用
  旧的 RAW 解码假设。
- `softloaf_raw_probe` 和 `softloaf_raw_provenance_matrix` 现在会输出更多
  RAW 诊断信息，包括 sensor hints、buffer shape、黑白场 level、尺寸和
  matrix source。
- 新增真实样本回归覆盖，包括 Sony packed full-color、Canon reduced RAW、
  Leica SL2 DNG、Phase One IIQ、Nikon HE 边界行为，以及 X3F fail-closed
  行为。

## Frames 工作流

- `Cmd/Ctrl + A`：全选所有完整 Frame。
- `Cmd/Ctrl + Shift + A`：取消全选。
- Shift 点击两个 Frame：选中两者之间的连续 Frame。
- 点击 Frames 面板空白区域：清空选择。
- 每个 Frame 卡片右上角现在都有删除按钮。
- Frames 旁边的进度条现在会显示有多少完整 Frame 已经可以用于
  Composite preview。

## Windows

- 改善 Windows 上 Qt Quick Controls 的样式兼容性。
- 更新 Windows staged app、安装包和 portable zip 的打包流程。
- 已在 303mini 上验证 Windows app 可以启动并保持响应。

## UI 调整

- Sensor 选择改为 Bayer / Monochrome 分段控件。
- 恢复紧凑的 SoftLoaf Trichrome 标题样式。
- 保留 TIFF 作为当前实现完整的导出路径，支持 8-bit/16-bit、ICC profile
  和已命名的目标色彩空间。

## 边界说明

- 这不是“所有 RAW 相机都已正确支持”的版本。
- DNG 导入仍通过 LibRaw 支持；DNG 导出仍未在本版本实现。
- Nikon HE/TicoRAW、Sony A7M5 `Compression=Next`、Foveon/X3F、
  pixel-shift/high-resolution RAW、monochrome RAW、cinema/special RAW
  仍不属于普通 RAW 支持范围，除非后续明确说明。
- Phase One IIQ 支持指的是 LibRaw 导入/解码支持，不代表 Capture One
  厂商配置文件或校正结果一致。
- RAW 色彩正确性仍不声明达到 RawTherapee parity；剩余差异记录在
  `docs/raw_correctness_report.md`。
- Windows 包如果没有配置 Authenticode 签名，仍可能触发 SmartScreen。
