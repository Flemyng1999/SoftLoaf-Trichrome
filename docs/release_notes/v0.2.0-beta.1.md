# v0.2.0-beta.1

SoftLoaf Trichrome first beta release. This build moves beyond the alpha line
with stronger export behavior, guarded RAW handling, provenance-aware caching,
and clearer release packaging.

This is still a beta release. It is suitable for broader testing of trichrome
import, preview, export, color-managed output, and guarded RAW workflows, but it
is not a full camera compatibility or RAW color-correctness guarantee.

## Changes

- Added structured RAW classification for Bayer/other CFA, X-Trans, packed RGB,
  packed four-color, monochrome, Foveon, and unknown `filters == 0`.
- Guarded default linear Rec.2020 RAW output so only Bayer/other CFA enters that
  path.
- Marked X-Trans, packed RGB, and packed four-color as camera-native
  fallback-only rather than silently labeling them correct Rec.2020 output.
- Marked monochrome, Foveon, and unknown `filters == 0` as unsupported for the
  default RAW path.
- Added RAW provenance records for raw class, policy, decode mode, fallback
  status, target color space, and stable signature.
- Added RAW provenance output to `softloaf_raw_probe`, including blocked/failure
  paths where LibRaw metadata can be inspected.
- Added `softloaf_raw_provenance_matrix` for local sample classification CSVs.
- Included RAW policy/provenance in preview cache identity and artifact
  identity.
- Added readable per-source RAW provenance fields to artifact build metadata.
- Improved export naming so exported frames use source stems and avoid
  overwrites.
- Continued color-managed export hardening with centralized color constants and
  ICC/profile regression coverage.

## Known Limitations

- RAW classification/provenance boundaries are protected, but this release does
  not claim full camera-family coverage.
- Local provenance smoke coverage for this release found CR2, RAF, DNG, 3FR,
  and RW2 samples, all classified as `bayer_or_other_cfa`; it did not cover
  X-Trans, packed full-color, monochrome, Foveon, or unknown `filters == 0`.
- RAW color correctness is not RawTherapee parity; remaining differences are
  documented in `docs/raw_correctness_report.md`.
- Windows packages may still trigger SmartScreen if Authenticode signing is not
  configured for this release.

## Packaging Notes

- macOS DMG packaging must pass bundle dependency resolution, code signing,
  notarization, and macOS 15.0 `minos` compatibility checks.
- Windows release publishes both installer and portable zip packages with Qt,
  OpenCV, LibRaw, Little CMS, and runtime dependencies.
